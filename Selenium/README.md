# 第六章 UI自动化测试

## 6.1 自动化测试概述

为了提高软件测试效率、减少人工重复操作，并验证系统核心功能模块的稳定性，本项目采用 Python + Selenium 实现 Web UI 自动化测试。

本次 UI 自动化测试以若依（RuoYi）后台管理系统作为测试对象，采用 Python 语言结合 Selenium 自动化测试框架进行实现，通过 Chrome 浏览器驱动程序模拟真实用户操作，对系统核心功能模块进行了自动化验证。测试内容主要涵盖系统登录、用户管理、角色管理、菜单管理以及通知公告管理等核心业务场景。

在测试过程中，自动化脚本能够自动完成页面访问、数据输入、按钮点击、页面跳转、表单提交以及结果校验等操作，并通过 Selenium 提供的元素定位与等待机制，对页面功能是否正常进行了验证。为了保证测试过程的完整性与可追溯性，每个测试场景均在操作前后分别进行截图保存，用于记录系统运行状态及测试执行结果。

本次自动化测试共设计并实现了 7 个典型业务场景，包括用户登录、用户新增、用户编辑、用户删除、角色管理、菜单管理以及通知公告管理。通过自动化执行，可以有效减少人工重复操作，提高测试效率，同时验证系统在常见业务流程下的稳定性与功能正确性。

自动化测试过程中，系统能够自动完成页面跳转、元素定位、表单填写、按钮点击、结果校验以及截图保存等操作。

## 6.3 自动化测试流程

**图 6-1 自动化测试流程图**

本次 UI 自动化测试首先启动若依（RuoYi）系统的前后端服务，确保被测系统能够正常运行。随后启动 Google Chrome 浏览器，并通过 Selenium 自动化框架访问系统登录页面，模拟用户输入账号密码完成登录操作。在系统成功进入后台后，自动化脚本按照预设测试流程依次执行各项业务场景，包括用户管理、角色管理、菜单管理以及通知公告管理等功能操作。在执行过程中，脚本通过元素定位与页面状态判断对功能结果进行自动校验，并在每个测试场景执行前后自动保存页面截图，用于记录测试过程与结果。全部测试完成后，系统最终输出自动化测试结果，为后续测试分析与实验报告编写提供依据。

## 6.4 自动化测试脚本设计

本项目采用 Python 语言编写 UI 自动化测试脚本，并结合 Selenium 自动化测试框架实现对若依（RuoYi）后台管理系统的功能验证。在测试实现过程中，通过 WebDriver 驱动 Chrome 浏览器完成页面访问与用户操作模拟，利用 XPath 对页面元素进行精准定位，并结合 WebDriverWait 显式等待机制提高页面加载与元素交互的稳定性。针对部分动态组件与按钮点击问题，测试脚本通过 JavaScript 实现页面元素的输入与点击操作，从而提升自动化执行成功率。同时，系统还集成了自动截图功能，在测试执行前后自动保存页面状态，用于记录测试过程与结果分析。此外，脚本中加入了异常处理机制，当测试过程中发生错误时能够自动捕获异常并保存错误截图，以便后续问题定位与调试。

系统主要封装了以下公共方法：

**表6-2 系统封装表**

|     |     |
| --- | --- |

## 6.7 本章小结

本章基于 Python 与 Selenium 自动化测试框架，对若依后台管理系统进行了 UI 自动化测试实践，重点覆盖了用户管理、角色管理、菜单管理以及通知公告等7个核心业务功能场景。在测试过程中，通过编写自动化脚本模拟用户在系统中的实际操作流程，实现了从登录、页面跳转到数据新增、修改与删除等一系列关键业务操作的自动验证，从而全面检验系统前端交互与后台功能的协同运行情况。

测试结果表明，系统各核心功能运行稳定，自动化测试脚本能够正常执行且具备较高的可靠性，页面元素定位准确，交互逻辑符合预期，同时截图记录与结果校验机制也能够有效辅助测试结果分析。通过本次实验，不仅验证了系统功能的正确性与稳定性，还进一步加深了对 Selenium 自动化测试框架、Web 元素定位技术以及 UI 自动化测试整体流程的理解与掌握，为后续软件测试与自动化测试相关实践奠定了良好基础。
